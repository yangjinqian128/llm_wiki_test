# 大页

大页是大于标准 4KB 页面的内存分配方式，减少 TLB miss 和缺页次数。

## 类型

1. 传统大页(hugetlb): 预分配固定池
2. [[透明大页]](THP): 动态合并

## 传统大页配置

### 内核配置
`CONFIG_HUGETLBFS`, `CONFIG_HUGETLB_PAGE`

### sysfs 接口
`/sys/kernel/mm/hugepages/`
```
hugepages-64kB/
hugepages-2048kB/  (2MB)
hugepages-1048576kB/ (1GB)
  ├── nr_hugepages      # 配置数量
  ├── free_hugepages    # 空闲数量
  └── surplus_hugepages
```

## mmap 使用

```c
// 使用默认大页
mmap(NULL, len, PROT_READ|PROT_WRITE,
     MAP_PRIVATE|MAP_ANONYMOUS|MAP_HUGETLB, -1, 0);

// 指定 64KB 大页
mmap(NULL, len, PROT_READ|PROT_WRITE,
     MAP_PRIVATE|MAP_ANONYMOES|MAP_HUGETLB|(16<<26), -1, 0);
```

## 页大小编码

`MAP_HUGE_2MB`: `21 << 26` (log2 页大小左移)

## 注意事项

- mmap 后需写入才真实分配
- 传统大页与 [[伙伴系统]] 无关
- 大页内存不计入普通内存统计

## 应用场景

- 数据库：减少 TLB miss
- DPDK：高性能网络
- 虚拟化：减少 EPT 映射层级
