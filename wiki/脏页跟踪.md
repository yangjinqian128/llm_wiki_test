# 脏页跟踪

脏页跟踪是虚拟化中追踪 Guest 内存修改的机制，用于 [[热迁移]]、快照等功能[^1]。

## KVM 脏页跟踪

开启时触发[^1]：

```
KVM_SET_USER_MEMORY_REGION (开启脏页跟踪)
  → kvm_mmu_split_huge_pages()
    → 拆分大页为 4K
  → kvm_stage2_wp_range()
    → 写保护所有 PTE
```

写保护后的写操作[^1]：

```
Guest 写被保护页 → Permission Fault
  → user_mem_abort()
    → force_pte = true
    → 按 4K 重新映射
    → 标记脏页
```

## 脏页位图

每个 [[RAMBlock]] 维护[^1]：

- `rb->bmap`: 主脏页位图，记录待传输脏页
- `rb->clear_bmap`: 辅助位图，记录已清理 KVM 脏页日志的 chunk

## SMMU HTTU 路径

[[HTTU]] 硬件自动标记脏页[^1]：

- DBM=1 的 writable PTE
- 硬件写操作自动清除 AP_RDONLY
- 软件遍历页表检测脏状态

## VFIO 路径

- VFIO type1: 软件 bitmap 在 vfio_dma 层[^1]
- iommufd S1: 硬件 DBM[^1]
