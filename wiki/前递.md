# 前递 (Forwarding)

流水线中解决数据冒险的技术，将计算结果直接传递给依赖指令，无需等待写回。

## 原理

数据冒险示例：

```
add x19, x0, x1   # x19在WB阶段写入
sub x2, x19, x3   # EX阶段需要x19
```

传统方式：sub停顿等待add写回

前递方式：
- add在EX阶段计算出x19值
- 直接前递给sub的EX阶段
- 无需等待WB阶段
- 流水线无停顿

## 实现需求

前递逻辑需要：
1. **冒险检测**: 检测前后指令是否存在数据依赖
2. **前递通路**: 从EX/MEM阶段到EX阶段的额外数据路径
3. **前递控制**: 选择前递数据还是寄存器堆数据

## 前递时机

关键计算：
- 不能出错：数据必须是正确结果
- 性能要高：尽早前递使流水线衔接

指导原则：尽量早前递

对于多拍执行指令：
- 后续指令可能后面几拍才用到结果
- 早前递使无依赖拍先执行
- 结果生成和使用无缝衔接

## 超标量扩展

超标量处理器：
- 多个执行单元并行
- 前递网络呈网状结构
- Issue queue的指令可能依赖多个执行单元结果
- 旁路网络复杂性增加
