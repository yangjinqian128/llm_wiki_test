# 寄存器重命名

将架构寄存器映射到更多物理寄存器，消除WAW/WAR假依赖。

## 目的

1. 消除假依赖，允许乱序执行
2. 支持投机执行结果暂存

## 基本映射

```
架构寄存器 → 物理寄存器
a0 → p0, a1 → p1, ...
```

## 数据结构

- **map表**：架构寄存器→当前物理寄存器映射
- **a_to_p map**：架构寄存器访问时的实际物理寄存器
- **free list**：空闲物理寄存器池
- **ROB**：保存指令的映射信息及前次映射

## 超标量rename

一拍rename多条指令需处理：
- RAW依赖：输入寄存器直接使用被依赖指令的物理寄存器
- WAW依赖：输出寄存器只写入最后一条的映射
- WAR依赖：不影响rename逻辑

## 恢复机制

| 方法 | 实现 |
|------|------|
| Checkpoint | 每条跳转指令保存map副本 |
| Walk | 按ROB反向恢复映射 |
| a_to_p map | 等待执行到分支点后拷贝 |
