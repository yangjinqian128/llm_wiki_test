# 两位饱和计数器

分支预测中的状态跟踪机制，根据历史跳转结果预测未来分支行为。

## 状态定义

计数器4个状态：
```
    strong   weak  weak  strong
       no taken     taken
   +-------------------------------+
   |     +-+   +-+   +-+   +-+     |
   |  +--|0|-->|0|-->|1|-->|1|--+  |
   |  |  | |   | |   | |   | |  |  |
   |  +->|0|-->|1|-->|0|-->|1|<-+  |
   |     +-+   +-+   +-+   +-+     |
   +-------------------------------+
```

- **00**: strong no taken
- **01**: weak no taken
- **10**: weak taken
- **11**: strong taken

## 预测逻辑

计数器值决定预测：
- 00/01: 预测no taken
- 10/11: 预测taken

强状态预测更稳定，弱状态更易改变。

## 状态转移

基于实际跳转结果更新：

| 当前状态 | 实际结果 | 新状态 |
|---------|---------|--------|
| 00 (strong no) | no taken | 00 |
| 00 (strong no) | taken | 01 |
| 01 (weak no) | no taken | 00 |
| 01 (weak no) | taken | 10 |
| 10 (weak taken) | taken | 11 |
| 10 (weak taken) | no taken | 01 |
| 11 (strong taken) | taken | 11 |
| 11 (strong taken) | no taken | 10 |

## 设计优势

单次预测失败不立即改变方向：
- strong taken首次失败仍预测taken
- 需连续两次失败才改变预测
- 提高预测稳定性，防止零星意外影响

## 应用场景

用于分支预测：
- 存储在PHT (Pattern History Table)
- 配合BHR做局部历史预测
- 每个BHR状态对应一个计数器
