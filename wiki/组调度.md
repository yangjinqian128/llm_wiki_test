# 组调度

组调度将若干线程组织到 group，作为调度实体参与调度。

## 与调度组的区别

- 组调度: 线程集合，调度单位
- [[调度组]]: CPU 划分，负载均衡单位

## 使用场景

1. cgroup: 控制用户/容器 CPU 资源
2. autogroup: 同 session 线程自动成组

## 数据结构

```c
struct task_group {
    struct cfs_rq *cfs_rq[NR_CPUS];  // 每CPU的cfs_rq
    struct sched_entity *se[NR_CPUS]; // 每CPU的se
    // ...
};
```

## 调度逻辑

```
pick_next_task_fair
  do {
    se = pick_next_entity(cfs_rq);
    cfs_rq = group_cfs_rq(se);  // 进入组内
  } while (cfs_rq);
```

层级选择：从根 cfs_rq 选出 group se，再进入 group 内 cfs_rq 继续。

## CPU cgroup 接口

`/sys/fs/cgroup/cpu/`
- `cpu.shares`: 权重分配
- `cpu.cfs_quota_us`: CPU 时间配额
- `cpu.cfs_period_us`: 配额周期

## 带宽控制

`cpu.cfs_quota_us` 在 `cpu.cfs_period_us` 内限制 CPU 使用时间，详见 [[CPU带宽控制]]。

## 时间分配

组获得的 CPU 时间按比例分配给组内任务，组的 se 参与上层调度。
