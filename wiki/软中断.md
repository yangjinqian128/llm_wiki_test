# 软中断

软中断是 Linux 内核模拟硬件中断行为的软件机制，在特定点检测并执行预设函数。

## 与硬中断区别

- 硬中断: CPU 硬件打断执行流
- 软中断: 软件层面模拟，检查标记触发

## 软中断类型

```
HI_SOFTIRQ        : 高优先级
TIMER_SOFTIRQ     : 定时器
NET_TX_SOFTIRQ    : 网络发送
NET_RX_SOFTIRQ    : 网络接收
BLOCK_SOFTIRQ     : 块设备
TASKLET_SOFTIRQ   : tasklet封装
SCHED_SOFTIRQ     : 调度负载均衡
HRTIMER_SOFTIRQ   : 高精度定时器
RCU_SOFTIRQ       : RCU处理
```

## 触发点

硬中断退出时检查 pending 软中断：
```
irq_exit
  -> __irq_exit_rcu
    if (local_softirq_pending())
      -> invoke_softirq
```

## 执行方式

1. 立即执行: 在当前上下文运行
2. 内核线程: 唤醒 `ksoftirqd` 线程

选择依据：
- `CONFIG_PREEMPT_RT`: 强制线程执行
- 是否有 `ksoftirqd` 运行

## 与抢占关系

软中断执行时关闭[[内核抢占]]：
- 防止软中断处理程序被抢占
- 保证软中断原子性执行
- `local_bh_disable()` 同时关闭软中断和抢占

## 控制接口

- `raise_softirq`: 触发软中断
- `local_bh_disable()`: 关闭软中断
- `local_bh_enable()`: 开启软中断

## tasklet

tasklet 是软中断的封装，新代码建议使用 tasklet 而非新增软中断类型。

## 内核线程

每个 CPU 一个 `ksoftirqd` 线程，处理负载较高的软中断场景。
