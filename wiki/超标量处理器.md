# 超标量处理器

每周期可取多条指令并行执行的处理器架构。

## 基本架构

```
前端(front-end): IF → ID
          ↓
发射逻辑(issue logic)
          ↓
后端(back-end): issue queue → EX → MEM → WB → commit
```

## 指令依赖

| 依赖 | 类型 | 解决方案 |
|------|------|----------|
| RAW | 真依赖 | 无法消除 |
| WAR | 假依赖 | 寄存器重命名 |
| WAW | 假依赖 | 寄存器重命名 |
| 控制 | 分支 | 分支预测 |

## 执行单元类型

- ALU(ADD/LOGIC)
- ALU(MUL/DIV)
- LOAD/STORE
- FP/SIMD

## 性能瓶颈

| 瓶颈 | 原因 |
|------|------|
| bad speculation bound | 分支预测不准导致频繁flush |
| back-end bound | ROB满反压前端 |
| front-end bound | 取指/译码瓶颈 |
