# 负载均衡

负载均衡是调度子系统动态将任务从高负载 CPU 迁移到低负载 CPU 的机制。

## 基本逻辑

```
time1:  core0    core1    core2    core3
           ||||    
           
time4:  core0    core1    core2    core3
           |       |       |       |
```

从最底层 [[调度域]] 开始，逐层向外均衡，高层级均衡间隔更长。

## 触发点

1. 时钟中断: `run_rebalance_domains`
2. fork/wakeup: 选择合适 CPU
3. new idle: idle CPU 主动拉任务
4. noHz idle: idle CPU 收 IPI 后均衡

## 核心流程

```
rebalance_domains
  for_each_domain(cpu, sd)
    if time_to_balance
      load_balance
        -> find_busiest_group
        -> find_busiest_queue
        -> detach_tasks
        -> attach_tasks
```

## 均衡判断

- local group 负载高于 busiest 不均衡
- busiest 有多个 idle CPU 不均衡
- busiest 只有一个任务不均衡
- 超过 `imbalance_pct` 才均衡

## 不均衡类型

- load: 按负载值迁移
- util: 按 CPU 利用率
- task: 按任务数
- misfit: 任务超出 CPU 能力

## 硬件影响

- SMT: 先填空物理核，再填剩余逻辑核
- 大小核: 重负载到大核
- NUMA: [[NUMA balancing]] 随线程迁移内存

## 调试接口

`/sys/kernel/debug/sched/debug`
