# 透明大页

透明大页(THP)是 Linux 自动将小页合并成大页映射的机制，减少缺页和 TLB miss。

## 与传统大页的区别

- 传统大页：预分配固定大小，与 [[伙伴系统]] 无关
- THP：动态合并，通常 2MB(order=9)，依赖伙伴系统

## sysfs 配置

### 使用范围
`/sys/kernel/mm/transparent_hugepage/enabled`
- always: 全系统使用
- madvise: 指定地址范围使用
- never: 关闭

### 生成方式
`/sys/kernel/mm/transparent_hugepage/defrag`
- always: 立即 stalloc 等待
- defer: 异步处理
- defer+madvise: madvise 立即处理

### khugepaged 参数
`/sys/kernel/mm/transparent_hugepage/khugepaged/*`

## 分配路径

缺页处理时直接分配大页：
```
__handle_mm_fault
  -> create_huge_pmd
    -> do_huge_pmd_anonymous_page
      -> alloc_hugepage_vma
```

## khugepaged 线程

扫描系统页面，将连续小页合并成大页：
```
khugepaged
  -> khugepaged_scan_pmd
    -> collapse_huge_page
```

## collapse 流程

1. 分配 2MB 连续页面
2. 清空 pmd，做 TLB invalidate
3. 复制小页内容到大页
4. 安装新 pmd 页表

同步点在 TLB invalidate，保证其他 CPU/设备的访问一致性。

## madvise 使用

```c
p = mmap(NULL, MEM_SIZE, PROT_READ|PROT_WRITE, 
         MAP_PRIVATE|MAP_ANONYMOUS, -1, 0);
madvise(p, MEM_SIZE, MADV_HUGEPAGE);
```

## 统计接口

- `/proc/vmstat`: thp_* 参数
- `/proc/meminfo`: AnonHugePages
- `/proc/PID/smaps`: 进程大页统计
