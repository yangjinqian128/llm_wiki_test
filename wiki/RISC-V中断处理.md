# RISC-V中断处理

## 概述

CPU中断机制允许外部信号打断当前执行流，跳转到预定义地址处理。

## 中断调度模型

```
               +----------------------+
               |                CPU   |
               |+--------------+      |
      -------->|| irq schedule |      |
               |+--------------+      |
               |                      |
               +----------------------+
```

## 中断目标识别

中断需要投递到特定的软件上下文，涉及：

- **target_irq_level**：中断目标特权级
- **current_cpu_run_level**：CPU当前运行特权级

### 处理策略

1. **目标=当前**：直接中断当前上下文
2. **目标>当前**：立即陷入高特权级处理
3. **目标<当前**：记录中断，等待切换到目标特权级

## 多实例处理

虚拟化场景下，中断目标是特定特权级上的软件实例：

- 需要识别当前实例是否为目标实例
- 非目标实例时，中断报告给高特权级管理

## RISC-V特权级模型

```
 +----+
 | VU |
 +----+
   |
 +-+--+           +----+
 | VS |           | U  |
 +----+           +----+
   |                |
   |     +----+     |
   +-----| HS |-----+
         +----+
           |
         +-+-+
         | M |
         +---+
```

## 中断类型

参见[[RISC-V特权级模型]]：

- **MEIP/SEIP**：外部中断（由中断控制器配置）
- **MTIP/STIP**：时钟中断
- **MSIP/SSIP**：软件中断（核间通信）

## 委托机制

通过medeleg/mideleg将中断/异常委托到S模式处理。
