# 多核启动

计算机系统有多个核时，多个核之间启动需要遵守一定的逻辑关系[^1]。

## 核心原则

虽然多核系统上每个核都可以独立运行程序，但多个核共享的资源需要串行配置和访问[^1]：

- 固件或内核的 BSS 段
- 固件的重定位
- 其他公共资源初始化

一般用一个核（主核）完成公共资源初始化，其他核（从核）在此基础上继续各自的初始化[^1]。

## 硬件视角

从 QEMU 角度看，多核启动就是多个核独立开始执行指令[^1]：

- 每个 vCPU 用一个线程模拟
- CPU 复位配置初始状态
- CPU 开始运行前需从 stopped 状态恢复

QEMU 和 [[OpenSBI]] 的接口[^1]：
- a0：当前核的 id（mhartid）
- a1：DTS 基地址
- a2：QEMU 和 OpenSBI 传递信息的内存地址

## 固件逻辑

[[OpenSBI]] 使用 lottery 算法选择主核[^1]：

```
多个核去抢做主启动核
→ 主核做公共资源初始化
→ 其他核（从核）等待
→ 主核完成初始化后，从核继续各自初始化
```

从核挂起方式[^1]：
- 汇编部分：wfi 指令
- C 代码部分：等待 coldboot 完成
- 主核通过 IPI 解开挂起的从核

## HSM 扩展

从核继续运行需要内核通过 [[SBI]] 的 HSM（Hart State Management）接口下发启动命令[^1]：

- `sbi_hsm_hart_start`：启动指定 hart
- 参数：hart id、执行起始地址（S mode）

从核的内核启动地址不是 `_start`，而是 `secondary_start_sbi`[^1]。

## 内核逻辑

Linux 内核 RISC-V 有两种多核启动实现[^1]：

### spinwait 方式

- 多核均直接进入内核
- 内核使用 lottery 算法选出主核
- 其他核 spinwait
- 不支持 CPU hotplug 和 sparse hartid
- 只用于只支持 M mode 或 BIOS 不支持 SBI HSM 扩展的情况[^1]

### ordered booting 方式

- 依赖 [[HSM]] 扩展
- 只有主核进入内核
- 内核通过 HSM 方法启动从核[^1]

启动流程[^1]：
```
start_kernel
  → setup_arch
    → setup_smp（挂接 SBI 回调函数）
  → kernel_init
    → smp_init
      → bringup_noboot_cpus
        → cpu_up
          → __cpu_up
            → start_secondary_cpu
              → sbi_cpu_start
```
