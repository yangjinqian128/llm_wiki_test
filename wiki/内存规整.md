# 内存规整

内存规整是通过迁移页面消除碎片，为连续内存分配腾出空间。

## 与内存迁移关系

[[内存迁移]] 包含：
1. 数据在物理内存搬移
2. 虚拟地址到物理地址重映射

内存规整侧重第一点，在伙伴系统内合并小块。

## 触发场景

- 分配连续大块内存
- [[透明大页]] 合并
- `alloc_contig_pages`

## 基本逻辑

```
L1拆借到L0:
  [块1'] [块2'] [块3'] [块4']
  
迁移块1'数据到其他页 -> 释放块1 -> 合并为大块
```

## 不可迁移页

某些页面标记为 unmovable，不能参与规整：
- 内核关键数据
- 被 mlock 锁定的页
- 特定驱动内存

## 触发函数

```
alloc_contig_pages
  -> compact_zone
    -> migrate_pages
```

## 调试接口

`/proc/sys/vm/compact_memory`: 触发系统级规整
