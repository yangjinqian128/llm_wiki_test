# 文件描述符引用计数

Linux内核通过`struct file`的`f_count`引用计数保证close与其他文件操作的同步。

## 创建文件

```c
filp_open
  → file_open_name
    → do_filp_open
      → path_openat
        → alloc_empty_file  // f_count初始化为1
```

## 文件操作

```c
ksys_ioctl
  → fdget
    → __fget_light
      → get_file_rcu_many  // f_count非0时才加1
  → fdput
    → fput
      → atomic_long_dec_and_test  // 减到0触发延迟释放
```

## 关闭文件

```c
close
  → __close_fd
    → spin_lock(&files->file_lock)  // 获取fd对应file
    → filp_close
      → fput  // 释放引用
    → spin_unlock
```

## 同步保证

- `__fget`检查`f_count`非0才增加引用
- close在锁内操作，确保不会访问已释放文件
- 引用计数减到0后延迟释放（delayed_fput_work）
