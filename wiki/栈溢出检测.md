# 栈溢出检测

ARM64 Linux 内核在异常向量表入口检测栈溢出[^1]。

## 检测原理

利用进程栈/IRQ 栈按 `THREAD_SIZE` 对齐的特性[^1]：

- 若 SP 的 `THREAD_SHIFT` 位为 1
- 说明 SP 已越界

## kernel_ventry 宏实现

```asm
add  sp, sp, x0          // sp' = sp + x0
sub  x0, sp, x0          // x0' = sp
tbnz x0, #THREAD_SHIFT, 0f  // 测试 THREAD_SHIFT bit
// 正常路径
...
0:  // 栈溢出路径：切换到 overflow_stack
```

## 溢出处理

切换到 per-CPU 的 `overflow_stack`[^1]：

```asm
adr_this_cpu sp, overflow_stack + OVERFLOW_STACK_SIZE, x0
// 检查是否已在 overflow stack 上
mrs  x0, tpidr_el0
sub  x0, sp, x0
tst  x0, #~(OVERFLOW_STACK_SIZE - 1)
b.ne __bad_stack  // 不在 → 真正的 bad stack
```
