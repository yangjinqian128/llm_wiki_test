# 流水线冒险

流水线执行中阻碍指令并行的问题，分为数据冒险、控制冒险和结构冒险。

## 数据冒险

读在写后的依赖，无法避免：

```
add x19, x0, x1   # x19在WB阶段才生效
sub x2, x19, x3   # EX阶段需要x19
```

### 解决方案

- **停顿**: sub在EX阶段停顿，等x19值得到
- **前递（forwarding）**: x19一算出就给sub指令

前递需要：
- 数据冒险检测逻辑
- 前递数据通路逻辑

## 控制冒险

流程控制指令导致PC跳变：

```
add x4, x5, x6
beq x1, x0, 40   # 需等x1、x0计算结果
or x7, x8, x9    # 可能选错了送入流水线
```

### 解决方案

- **分支预测**: 预测分支方向和目标
- **延迟分支**: 编译器填充有用的指令
- **分支延迟槽**: 预测失败时减少损失

控制冒险一定会存在，缓解办法是提高分支预测准确率。

## 结构冒险

硬件资源冲突，多条指令同时需要同一资源：
- 多个执行单元可缓解
- 资源复制可避免

现代CPU多发射设计缓解结构冒险。

## 依赖类型

指令间寄存器依赖：
- **RAW (Read After Write)**: 真依赖，无法消除
- **WAR (Write After Read)**: 假依赖，寄存器重命名消除
- **WAW (Write After Write)**: 假依赖，寄存器重命名消除

Load/Store地址依赖：
- RAW/WAR/WAW都是真依赖
- WAR: 先读后写，store先执行逻辑错
