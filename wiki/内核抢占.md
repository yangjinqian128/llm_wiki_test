# 内核抢占

内核抢占是指线程在内核执行时被其他线程打断，抢占 CPU 资源的行为。

## 抢占类型

1. 主动让出 CPU: `wait_for_completion` 等
2. 被动抢占: tick 中断调度、中断触发调度

## 控制接口

```c
// 禁止抢占
preempt_disable()  // preempt_count++

// 开启抢占
preempt_enable()   // preempt_count--, 可能触发调度

// 是否可抢占
preemptible() = (preempt_count == 0 && !irqs_disabled())
```

## 触发点

1. `preempt_enable`: 开抢占时检查调度
2. 中断返回内核: 恢复内核执行前检查

## 注意事项

- 禁抢占不关中断：中断可进入但返回后不调度
- 关中断时不可抢占
- preempt_count 是引用计数

## 使用场景

保护 per-CPU 变量：
```c
get_cpu_ptr(var)     // preempt_disable()
put_cpu_ptr(var)     // preempt_enable()
```

## 实时性

`CONFIG_PREEMPT_RT` 开启后，更多位置可抢占，提高实时性。
