# 硬件队列设计

硬件队列驱动的设计模式，考虑高性能和不溢出条件。

## 队列模型

```
N 个 SQ（发送队列） ──┐
N 个 CQ（完成队列） ──┼── N 个 QP（队列对）── 1 个 EQ（事件队列）
```

| 队列 | 说明 | 指针更新 |
|------|------|----------|
| SQ | 用户填入请求 | 软件 tail，硬件 head |
| CQ | 硫件填入完成 | 硬件 tail，软件 head |
| EQ | 中断事件 | 硬件 tail，软件 head |

## Doorbell 操作

- SQ doorbell：更新 SQ tail
- CQ doorbell：更新 CQ head
- EQ doorbell：更新 EQ head（同时打开中断屏蔽）

## 硬件框图

```
         doorbell                 irq
    ┌────────┐                 ┌────┐
    │  SQ    │──→ Core ──→     │ EQ │
    │  CQ    │←────────        │    │
    └────────┘                 └────┘
```

## CQ 溢出条件

SQ 不溢出 → CQ 无中断也能不溢出。

中断处理：先更新 CQ head，再更新软件 SQ head。

否则新 SQ 请求可能产生 CQE 导致 CQ 溢出。

## EQ 溢出条件

EQ 深度 ≥ 并发 CQ 数 N。

处理 n 个 EQE 后放开 ≤ n 个 CQ → EQ 通道。

实际实现需增加 EQ 深度到 N+n。

## 中断汇聚优化

```c
while (轮询已完成 n 个 eqe) {
    处理 eqe 对应的 cqe;
    打开 cq 到 eq 的通道;
}
更新 eq head 到原来 eq_head + n;
```

每次更新 EQ head 打开中断屏蔽，汇聚处理减少中断次数。

## CQE 汇聚处理

一个 EQE 至少对应一个 CQE，可汇聚 k 个 CQE 一起处理后再放开 CQ → EQ 通道。
