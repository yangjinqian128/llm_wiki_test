# 控制依赖

控制依赖是指条件分支对后续访存指令的排序约束。

## 读-写控制依赖

有效：
```c
q = READ_ONCE(a);
if (q) {
    WRITE_ONCE(b, 1);  // 不乱序到读 a 之前
}
```

条件分支阻止写提前执行。

## 读-读控制依赖

无效，需要读屏障：
```c
q = READ_ONCE(a);
if (q) {
    p = READ_ONCE(b);  // 可能乱序！
}
```

CPU 可能预读 b，需要 `smp_rmb()`。

## 编译器风险

编译器可能优化掉控制依赖：
```c
// 两分支写相同值
if (q) { WRITE_ONCE(b, 1); }
else    { WRITE_ONCE(b, 1); }

// 编译器可能提取公共表达式：
WRITE_ONCE(b, 1);  // 乱序！
```

解决方案：使用 `smp_store_release()` 或写不同值。

## 总结

- 读-写控制依赖有效
- 读-读需要读屏障
- 写-写需要写屏障
- 防止编译器优化掉条件
