# 异常向量表

异常向量表是 ARM64 架构处理异常的入口机制，定义了 16 个向量槽位对应不同异常类型和发生上下文[^1]。

## 硬件背景

ARM64 定义 4 种异常类型 × 4 种发生上下文 = 16 个向量槽位[^1]：

| | Sync | IRQ | FIQ | Error |
|---|---|---|---|---|
| **EL1t** (SP_EL0) | ✓ | ✓ | ✓ | ✓ |
| **EL1h** (SP_EL1) | ✓ | ✓ | ✓ | ✓ |
| **EL0 64-bit** | ✓ | ✓ | ✓ | ✓ |
| **EL0 32-bit** | ✓ | ✓ | ✓ | ✓ |

向量表基地址存放在 [[VBAR_EL1]] 寄存器，每条向量占 128 字节，整个表 2KB 对齐[^1]。

## 三层实现

Linux 内核异常向量表分三层实现[^1]：

1. **汇编层向量表入口** (`entry.S`) - `kernel_ventry` 宏
2. **汇编层 handler 分发** (`entry.S`) - `entry_handler` 宏
3. **C 层处理** (`entry-common.c`) - 按 ESR_EL1 的 EC 码分发

## Spectre 缓解

系统维护三套向量表应对 Spectre-BHB 攻击和 [[KPTI]][^1]：

| 向量表 | 位置 | 用途 |
|--------|------|------|
| `vectors` | `.entry.text` | 默认完整向量表 |
| `tramp_vectors` | `.entry.tramp.text` | KPTI 场景 |
| `__bp_harden_el1_vectors` | `.entry.text` | EL1 BHB 缓解 |
