# 伙伴系统

伙伴系统是 Linux 内核内存管理中用于缓解内存碎片的核心分配算法。

## 核心思想

将内存按不同大小的块组织，小块内存在特定内存区域分配，大块内存在更大区域分配，避免小内存块分散导致大内存分配困难。

## 基本逻辑

1. 内存按 order 分级，每个 order 对应 2^n 个连续页面
2. 每个 order 维护独立的空闲链表
3. 分配小内存时从大 order 拆分
4. 释放时检查伙伴是否空闲，若空闲则合并

## 伙伴查找算法

```c
// 伙伴的 PFN 计算
buddy_pfn = page_pfn ^ (1 << order)
```

简单地将 PFN 的对应位反转即可找到伙伴。

## 与内存迁移的关系

[[内存迁移]] 可以缓解碎片：迁移小页数据后释放，与伙伴合并成大块。虚拟内存的地址重映射是内存迁移的基础。

## 与大页的关系

透明大页(THP)需要从 order=9 的内存域分配，当内存不足时需要通过内存迁移腾出空间。传统大页与伙伴系统无关。

## 关键函数

- `alloc_pages`: 核心分配入口
- `__rmqueue`: 从空闲链表取页
- `__free_one_page`: 释放并合并伙伴

## 调试接口

`/proc/buddyinfo` 显示各 order 的空闲页面数。
