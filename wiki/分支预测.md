# 分支预测

超标量处理器对分支指令目的地址的预测机制。

## 基本逻辑

1. 预测分支目的地址
2. 从预测地址fetch指令投机执行
3. 分支结果确认后：正确→提交，错误→flush

## 分支类型

| 类型 | 特点 |
|------|------|
| 直接跳转 | 目的地址编码在指令中 |
| 条件跳转 | 根据条件决定是否跳转 |
| 间接跳转 | 目的地址在寄存器中 |

## 条件跳转预测

**局部历史（BHR）**：
```
BHR: 0001 → PHT(Pattern History Table)
       ↓
两位饱和计数器 → 预测taken/not taken
```

**全局历史**：BHR存放程序执行路径上跳转历史。

## 间接跳转预测

- **BTB**（Branch Target Buffer）：保存跳转目的地址
- **RAS**（Return Address Stack）：函数返回地址栈

详见[[间接跳转预测]]，包括：
- BTB 预测机制
- RAS 预测机制
- 与直接跳转预测的区别

## 两位饱和计数器

```
strong no ← weak no ← weak taken ← strong taken
```

两次预测失败后改变预测方向。
