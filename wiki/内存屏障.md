# 内存屏障

内存屏障是强制内存操作排序的机制，防止编译器和 CPU 乱序。

## 为什么需要

CPU 和编译器可能乱序执行：
- 不同地址的 load/store 可乱序
- 优化可能改变指令顺序

## 四种基本类型

1. **写屏障**: 屏障前写操作先完成
2. **读屏障**: 屏障前读操作先完成
3. **通用屏障**: 读写在屏障前后有序
4. **地址依赖屏障**: 依赖读有序(已废弃)

## Linux 内核 API

```c
smp_wmb()       // 写屏障
smp_rmb()       // 读屏障
smp_mb()        // 通用屏障
barrier()       // 编译器屏障
```

## ACQUIRE/RELEASE

单向屏障，详见 [[ACQUIRE语义]] 和 [[RELEASE语义]]。

## 编译器屏障

防止编译器优化：
```c
a = *x;
barrier();  // 编译器不合并前后读
b = *x;
```

## READ_ONCE/WRITE_ONCE

防止编译器优化且保证原子访问。

## 应用场景

- 锁操作
- [[RCU]] 读侧
- device MMIO
- 多核同步
