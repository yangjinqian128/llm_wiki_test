# 锁机制

[[锁机制]] 用于保护并发访问的数据结构，确保数据一致性[^1]。

## 锁的类型

- **[[spinlock]]**：死循环等待获取锁，适用于不可 sleep 的场景和短临界区[^1]
- **[[mutex]]**：获取锁失败后 sleep，适用于可 sleep 场景[^1]
- 读写锁版本：区分读/写操作的锁[^1]

## 死锁场景

典型死锁模式：
1. **交叉加锁**：两把锁在不同线程中反向加锁[^1]
2. **重复加锁**：同一执行流重复加同一把锁[^1]

## 避免死锁

解决交叉加锁：第二把锁使用 try 锁，失败则释放第一把锁[^1]。

## 调试方法

Linux 内核死锁检测选项：
```
Kernel hacking -> Debug Lockups and Hangs
```
[^1]

检测会报告潜在死锁位置，但会影响性能[^1]。

## 性能影响

锁的性能开销本质是执行流相互等待[^1]。

提高性能需解开执行流间的依赖[^1]。
