# 阿姆达尔定理

阿姆达尔定理刻画的是优化系统的一部分对整个系统的影响。

## 核心公式

系统总运行时间为 T，由 T1 和 T2 组成：
```
T = T1 + T2
```

如果 T1/T = a，那么 T = aT + (1-a)T。如果优化 T1，把 T1 优化到 T1/k，总的运行时间为：
```
T_new = aT/k + (1-a)T = (a/k + (1-a))T
```

总时间优化的倍数（加速比）为：
```
S = T/T_new = 1/(a/k + (1-a))
```

## 理论极限

当 k 趋向无穷大（即 T1 被完全优化），整个系统的优化倍数最大为：
```
S = 1/(1-a)
```

取具体值：如果 a = 1/4，整个系统的优化倍数就是 4/3。[^1]

## 并行计算视角

将公式进一步具体化：
- T1 是整个系统中可以并行的部分，增加 CPU 核数可以按比例减少这部分时间
- k 表示优化并行部分的 CPU 核数
- T2 表示系统的串行部分

**关键结论**：不断增加 CPU 核数，系统整体的优化倍数上限为 `S = 1/串行部分比例`[^1]

例如：
- 串行部分比例 1/4 → 上限是 4 倍
- 串行部分比例 1/2 → 上限是 2 倍

## 测试验证

可通过测试程序验证该定理：提前准备一组任务，多个线程不断从中取出任务执行。每个任务分并行部分和串行部分，当一个线程执行任务的串行部分时需要加上全局锁保护[^1]

测试参数：
- serial_ratio：调节串行和并行部分的比例
- working_time：测试持续时间
- pthread_num：CPU 核数
- work_num：完成任务个数

测试结果显示串行比例 50% 和 80% 时，最大加速比分别是 2 和 1.25，与理论分析一致[^1]
