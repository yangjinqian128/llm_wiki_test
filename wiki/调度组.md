# 调度组

调度组是 [[调度域]] 内进行负载均衡的基本单位。

## 与组调度的区别

- 调度组: CPU 的划分，用于负载均衡
- 组调度: 线程的集合，作为调度实体参与调度

## 组织方式

每个 domain 包含多个 group：
```
NUMA domain (cpu0-15)
  +-- group(cpu0-7)
  +-- group(cpu8-15)

MC domain (cpu0-7)
  +-- group(cpu0-3)
  +-- group(cpu4-7)
```

## 负载均衡流程

1. 找最繁忙 group
2. 从 group 内找最繁忙 CPU
3. 迁移任务到 local group

## 统计信息

- `group_load`: 组内各 CPU load_avg 之和
- `group_util`: CPU 利用率之和
- `group_runnable`: runnable_avg 之和
- `sum_nr_running`: 线程数之和

## group_type 分类

- `group_misfit_task`: 任务超出 CPU 能力
- `group_asym_packing`: 不对称打包
- `group_smt_balance`: SMT 均衡
- `group_overloaded`: 负载超算力
- `group_fully_busy`: 算力刚好满足
- `group_has_spare`: 算力有空余
