# BHR (Branch History Register)

分支历史寄存器，记录分支指令的taken/not taken历史，用于分支预测。

## 功能

对于条件跳转指令：
- 发生跳转记1
- 不发生跳转记0
- 跳转历史是0/1序列

BHR记录这个序列：
- 最大bit数决定可完美预测的"循环节"大小
- 与跳转序列规律位数相同时可做完美预测

## 局部历史预测

局部历史信息条件跳转预测方法：
- 每个分支指令对应一个BHR
- BHR值对应PHT (Pattern History Table)中的两位饱和计数器
- 根据BHR值查PHT得到预测结果

示意图：
```
BHR:           PHT
+------+       +------+                             
| 0001 | ----> | 0000 | -> 0 (no taken)
+------+       | 0001 | -> 1 (taken)
               | 0010 | -> 0
               | 0100 | -> 0
               | 1000 | -> 1
               +------+
```

## 示例分析

序列`000100010001...`：
- 四个状态循环
- BHR为4bit时，每个状态下一次跳转确定
- 可做完美预测

若BHR为5bit：
- Pattern: `00010/00100/01000/10001`
- 仍可完美预测
- PHT未用entry增多

## 全局历史扩展

全局历史预测：
- BHR存放程序动态执行路径上之前跳转指令的历史
- 预测根据"全局"跳转信息
- 受物理实现限制，BHR有位数限制
- 非理想全局信息
