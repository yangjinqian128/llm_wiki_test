# 内存回收

内存回收是 Linux 内核将物理页面释放回 [[伙伴系统]] 的机制。

## 基本逻辑

虚拟地址断开映射(munmap)与物理页面释放是正交的。内核只保证虚拟地址数据正确，物理页面管理独立进行。

## 回收类型

1. 直接回收：缓存页、无用内存直接释放
2. 需保存回收：
   - 文件页：写回 backend 文件
   - 匿名页：swap 到 swap 分区
   - shrink：压缩处理

## 核心流程

```
alloc_pages (慢速路径)
  -> __alloc_pages_direct_reclaim
    -> shrink_zones
      -> shrink_lruvec
        -> shrink_list
```

## kswapd 线程

每个 NUMA 节点一个 kswapd 内核线程，检测内存水线并触发回收。

## 状态管理

[[LRU链表]] 维护页面冷热状态，回收从 inactive LRU 选择页面。

## 触发接口

- `madvise(MADV_COLD/MADV_PAGEOUT)` 主动回收
- `alloc_contig_pages` 分配连续内存时回收
