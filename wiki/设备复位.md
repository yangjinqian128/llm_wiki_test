# 设备复位

设备从错误状态恢复到可运行状态的操作，保留用户配置。

## 复位类型

| 类型 | 说明 | 影响范围 |
|------|------|----------|
| 全局复位 | 整个设备复位 | 所有 PF/VF |
| Function 复位 | 单个 PF/VF 复位 | 仅该 Function |
| FLR | Function Level Reset | PCIe FLR |
| 部件复位 | 队列等部件复位 | 仅该部件 |

## 复位特点

复位后状态 ≠ 上电初始状态：

- **保留用户配置**：VF 数量、队列资源等
- **恢复运行状态**：清除错误，恢复可用

## 配置保存恢复

| 复位类型 | 配置位置 | 处理方式 |
|----------|----------|----------|
| 全局复位 | 设备全局寄存器/PF | 需保存恢复业务配置 |
| PF FLR | PCIe 协议规定 VF 被 disable | PCIe 总线驱动恢复 VF 数量 |

PF FLR 流程中，PCIe 总线驱动保留 VF 数目配置，复位后重新 enable VF。

## 带流量复位

复位时设备可能仍有流量运行，需先停止工作部件。

### 理想流程（有 PF/VF 通知机制）

```
PF 欲全局复位 → 通知 VF → VF 停止工作 → VF 通知 PF → PF 复位
```

### 无硬件通知机制的补救

依赖硬件行为：

- VF 在全局复位时读返回全 1、写被丢弃
- 软件发送请求加定时器
- 超时判断可能发生全局复位

**风险**：读全 1 可能误判，超时后判断复位完成困难。

## VF 在虚拟机中复位

全局复位时 VF 在虚拟机中使用，重新 enable 的 VF 是否可用需考虑。
