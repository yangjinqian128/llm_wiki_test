# 异步通知

Linux 内核向用户态进程主动发送信号的机制，通过 SIGIO 信号实现。

## 内核-用户态通知机制

| 机制 | 说明 |
|------|------|
| eventfd | 事件通知 |
| netlink | 网络消息 |
| 异步通知 | 设备文件信号 |

## 驱动实现

### fasync 回调

```c
int test_fasync(int fd, struct file *file, int mode)
{
	return fasync_helper(fd, file, mode, &async_queue);
}
```

注册 `fasync_struct`，内核分配结构体并返回地址。

### 发送信号

```c
void test_trigger_sigio(struct timer_list *tm)
{
	kill_fasync(&async_queue, SIGIO, POLL_IN);
}
```

向 fd 绑定的进程发送 SIGIO 信号。

## 用户态设置

```c
ret = fcntl(fd, F_SETOWN, getpid());  // 绑定进程

fcntl(fd, F_SETFL, fcntl(fd, F_GETFL) | FASYNC);  // 启用 fasync
```

## 信号处理

不同 shell 可能 mask SIGIO，需要 unmask：

```c
sigset_t set;
sigemptyset(&set);
sigaddset(&set, SIGIO);
sigprocmask(SIG_UNBLOCK, &set, NULL);
```

## 信号响应时机

信号标记在进程 signal pending 表，从内核态切回用户态时扫描响应。

即使进程无系统调用，内核调度也会触发用户态切换，确保信号响应。
